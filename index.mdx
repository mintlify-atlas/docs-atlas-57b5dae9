Mazinger Dubber is a Python CLI and library that takes a video URL or local file and produces a fully dubbed audio or video output — automatically transcribing speech, translating subtitles, cloning the speaker's voice, and assembling the final timeline. Every stage runs independently or as part of the full pipeline, and interrupted runs resume from where they left off.

## How it works

Mazinger chains ten stages into a single pipeline. Each stage can also run independently via its own sub-command.

<Steps>
  <Step title="Download and extract audio">
    Fetch the video from a URL (YouTube and more via yt-dlp) or ingest a local file. Extract the audio track for transcription.
  </Step>
  <Step title="Transcribe and analyze">
    Convert speech to SRT subtitles using OpenAI Whisper API, faster-whisper, WhisperX, or MLX Whisper. An LLM picks key frames and generates a structured content summary (title, key points, keywords) for translation context.
  </Step>
  <Step title="Translate with duration awareness">
    Translate subtitles into the target language using duration-aware word budgets so the dubbed speech fits the original timeline. Re-segmentation merges fragments and splits oversized entries for readability.
  </Step>
  <Step title="Synthesize and assemble">
    Clone the speaker's voice and synthesize speech for every subtitle entry using Qwen3-TTS, Chatterbox, or MLX TTS. Assemble segments on the original timeline with optional tempo adjustment, loudness matching, and background audio mixing. Optionally burn styled subtitles into the output video.
  </Step>
</Steps>

## TTS engines

Mazinger supports three TTS backends — pick the one that matches your hardware and workflow.

| Engine | Hardware | Key strength |
|--------|----------|-------------|
| **Qwen3-TTS** (default) | CUDA GPU | Multilingual, requires voice sample + transcript |
| **Chatterbox** | CUDA GPU | Emotion and pacing control, audio-only reference |
| **MLX TTS** | Apple Silicon | Native M-series performance, no CUDA needed |

<Note>
  Qwen, Chatterbox, and MLX require different dependencies and cannot share the same Python environment. Create a separate virtual environment for each engine.
</Note>
